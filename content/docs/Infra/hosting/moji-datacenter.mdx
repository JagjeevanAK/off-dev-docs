## Network Configuration

<Callout type="warn">
The server currently has **IPv6-only** public connectivity. No public IPv4 address is available.
</Callout>

<Card title="Network Access" description="IPv6-only server with proxy access options">

### Connection Details

<Steps>
  <Step title="IPv6 Direct Access">
    Server IPv6 address: `2a06:c484:5::45`
  </Step>
  
  <Step title="IPv4 Proxy Access">
    Connect via proxy server: `45.147.209.254` (has public IPv4)
  </Step>
  
  <Step title="VM/Container Access">
    Use individual VM/CT IPv6 addresses for direct connection
  </Step>
</Steps>

### Access Methods

<Tabs items={["IPv6 Direct", "IPv4 Proxy", "VM Access"]}>
  <Tab value="IPv6 Direct">
    For clients with IPv6 connectivity:
    ```bash title="Direct IPv6 Connection"
    ssh user@2a06:c484:5::45
    ```
  </Tab>
  
  <Tab value="IPv4 Proxy">
    For IPv4-only clients:
    ```bash title="Proxy Connection"
    ssh user@45.147.209.254
    # Then connect to internal server
    ```
  </Tab>
  
  <Tab value="VM Access">
    Each VM/Container has its own IPv6 address for direct access when deployed.
  </Tab>
</Tabs>

</Card>
