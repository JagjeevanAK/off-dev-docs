## Customization

### Styling

<Callout type="tip">
We added a lot of `part` attributes to the components, to allow you to customize the look and feel of the components.
</Callout>

<Card title="CSS ::part() Documentation" href="https://developer.mozilla.org/en-US/docs/Web/CSS/::part">
See ::part() attribute documentation on MDN
</Card>

### Translations

<Callout>
We only translated basic messages and most labels can generally be overridden using slots inside web components, where your own translation framework might be used (be it in JavaScript, or through your template engine or any technique).
</Callout>

<Callout type="warning">
If you need to override current translations, you might clone this project, change translations in xliff files and regenerate the bundle.
</Callout>

## Main Components

<Callout>
These are the components you will certainly use to build your interface. Of course none are mandatory and you can pick and choose the components you need.
</Callout>

<Callout type="tip">
Pay attention to the `search-name` attribute which must correspond to the `name` attribute of the search bar. If you do not specify it, it will be the default one. You'll need it if you mix multiple search bars in the same page.
</Callout>
