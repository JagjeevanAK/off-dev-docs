# Ingredient Detection

</Cards>

We developed a sequence tagging model to automatically extract ingredient lists from photos of product packaging.

## Model Overview

<Callout type="info">
  This model solely relies on the text content of the image, and does not require any spatial information. It is trained to detect ingredient lists in a sequence tagging format, where each token (word) is classified as either part of an ingredient list or not.
</Callout>

**Model Architecture:**
- **Base Model**: Fine-tuned from [xlm-roberta-large model](https://huggingface.co/FacebookAI/xlm-roberta-large)
- **Format**: Uses the IOB format for entity classes
- **Entity Classes**: Single `ING` entity class for ingredient lists, and `O` class for non-ingredient tokens

## Dataset

The model was trained on ~5000 texts extracted from product packaging, that were annotated semi-automatically. 

<Callout type="info">
  Look at [the dataset page](https://huggingface.co/datasets/openfoodfacts/ingredient-detection) on Hugging Face for more details.
</Callout>

## Robotoff Integration

### Inference and Post-processing

<Steps>
  <Step>
    ### Model Serving
    
    The model was exported to ONNX and is served by Triton server. The model integration in Robotoff can be found in the `robotoff.prediction.ingredient_list` module.
  </Step>

  <Step>
    ### Entity Aggregation
    
    After the individual token prediction, we aggregate the entities by merging the tokens that have the same entity class.
  </Step>

  <Step>
    ### Language Detection and Parsing
    
    We detect the language associated with the ingredient list using our language detection model (currently based on fasttext), and parse the ingredient list using the Product Opener ingredient parser. This allows us to know how many ingredients in the ingredient list are recognized (or not) by Open Food Facts.
  </Step>
</Steps>

### Prediction Data Fields

We add 7 fields in the image prediction data field:

<Cards>
  <Card title="lang" description="Language prediction with ISO 639-1 code and confidence score" />
  <Card title="ingredients_n" description="Total number of ingredients in the ingredient list" />
  <Card title="known_ingredients_n" description="Number of ingredients recognized by Open Food Facts" />
  <Card title="unknown_ingredients_n" description="Number of ingredients not recognized by Open Food Facts" />
  <Card title="fraction_known_ingredients" description="Fraction of recognized ingredients (between 0 and 1)" />
  <Card title="ingredients" description="List of parsed ingredients with detailed metadata" />
  <Card title="bounding_box" description="Relative coordinates of the ingredient list in the image" />
</Cards>

### Integration Process

For every new uploaded image, the model is run on this image. As for all computer vision models, we save the model prediction in the `image_prediction` table.

<Callout type="info">
  See function `robotoff.workers.tasks.import_image.extract_ingredients_job` for implementation details.
</Callout>

If some entities (i.e. ingredient lists) are detected, we create a `Prediction` in DB using the usual import mechanism, under the type `ingredient_list`.

<Callout type="note">
  See [this page](/docs/Robotoff/explanations/predictions.mdx) for more details about the import mechanism.
</Callout>

### Insight Generation Conditions

We only create an insight if the following conditions are met:

1. There is no ingredient list for the language detected in the extracted ingredient list
2. 60% or more of the ingredients in the extracted ingredient list are recognized by Open Food Facts as ingredients (using the Product Opener parser)

<Callout type="info">
  The insight has a null `value` and the detected language as `value_tag`. See `IngredientDetectionImporter.generate_candidates` for implementation details.
</Callout>

## Annotation Process

To annotate the insight, use the usual route:

```http title="Annotation endpoint"
POST https://robotoff.openfoodfacts.org/api/v1/insights/annotate
```

### Annotation Types

<Cards>
  <Card title="Validate As-Is" description="annotation=1 - User validates the ingredient list without changes" />
  <Card title="Update Ingredient List" description="annotation=2 - User submits an updated ingredient list with corrections" />
  <Card title="Reject Prediction" description="annotation=0 - User rejects the prediction" />
</Cards>

<Callout type="info">
  **Request Body Guidelines:**
  - **If updating**: Pass `annotation=2` and include `data` with the updated ingredient list
  - **If validating as-is**: Pass `annotation=1` without the `data` field
  - **If rejecting**: Pass `annotation=0` without the `data` field
</Callout>

### Post-Annotation Processing

If the insight is validated, we select the source image as an ingredient image for the detected language (if needed), with the right cropping and image orientation. 

<Callout type="note">
  If the user submitted an updated ingredient list, we select the image without cropping as we don't know the correct crop associated with the updated ingredient list.
</Callout>
